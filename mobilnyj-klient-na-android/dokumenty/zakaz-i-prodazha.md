Документы Заказ и **Продажа** полностью идентичны по интерфейсу. Предназначены для создания заказа или расходной накладной. Обычно это основной документ, создаваемый торговым агентом. Отключить или переименовать кнопку документа можно в [настройках](https://mobi-c.ru/help/mobimobimi8_journalagentssetting.htm) агента в обработке Моби-С.

| Флаг "Пр" | Функционал флага зависит от конфигурации 1С. Обычно используется для отражения документа в бух. и нал. учетах. |
| Фирма | Фирма, от которой мы работаем. При выборе клиента или договора фирма может автоматически измениться. Это происходит, если у клиента задана фирма по умолчанию. |

Договор Авто будет обработан при выгрузке документа в 1С, обычно вместо него в документ подставляется договор по умолчанию для данного клиента. При выборе договора может измениться фирма, тип цены и скидка. |
| Тип цен | Выбор типа цены. |
| Скидка | Выбор скидки. Скидка действует на весь документ в целом. |

После выгрузки документа происходит быстрое обновление остатков ([FastRests](https://mobi-c.ru/help/mobifastrests.htm)), сообщений ([Messages](https://mobi-c.ru/help/mobimessages.htm)), списка заданий ([FastTasks](https://mobi-c.ru/help/mobifasttasks.htm)), GPS трек ([GPS](https://mobi-c.ru/help/gps2.htm)) и выгружается лог работы ([ErrorLog](https://mobi-c.ru/help/mobierrorlog.htm)). |
| Кнопка "Выгрузить в фоне" | Сохраняет и закрывает документ. Документ выгрузится в 1С при следующем **фоновом** сеансе связи, которые осуществляются каждые 15 минут. При фоновой выгрузке происходит быстрое обновление остатков ([FastRests](https://mobi-c.ru/help/mobifastrests.htm)), сообщений ([Messages](https://mobi-c.ru/help/mobimessages.htm)), списка заданий ([FastTasks](https://mobi-c.ru/help/mobifasttasks.htm)), GPS трек ([GPS](https://mobi-c.ru/help/gps2.htm)) и выгружается лог работы ([ErrorLog](https://mobi-c.ru/help/mobierrorlog.htm)). |

В заголовке отображается итоговая сумма документа и процент наценки, с учётом выбора товара на вкладке **Товары**.

После ввода всех данных в документ можно:

1.  **Сохранить** документ в **Журнале**. Для этого нужно в документе нажать аппаратную или программную кнопку Назад. В диалоговом окне будет предложено сохранить документ. В **Журнале** вы сможете просмотреть, изменить или удалить этот документ. Для выгрузки документа выполните запрос **Выгрузка документов** или нажмите кнопку выгрузки в самом документе.
2.  **Выгрузить** документ в офис. Для этого нажмите круглую **зеленую** кнопку на первой вкладке документа. После выгрузки документа статус выгрузки можно посмотреть в **Журнале**.
3.  **Отправить на фоновую выгрузку**. Для этого нажмите круглую **синюю** кнопку на первой вкладке документа. Сохраняет и закрывает документ. Документ выгрузится в 1С при следующем сеансе фоновой связи. При наличии мобильного интернета, мобильный клиент каждые 15 минут автоматически выполняет попытку на выгрузку документов в офис.

### Особенности работы с выбором типа цен

В Моби-С на все или отдельные товары можно разрешить редактирование цены и скидки в карточке товара.

Где настроить: **Настройки агента → вкладка Цена и скидки**.

**Поведение при смене типа цены в зависимости от настроек**

Если для всех товаров запрещено редактирование цены: изменение цены будет применяться к всем товарам.

Если для товаров разрешено редактирование цены:

-   при выборе другого типа цены в списке появляется диалог выбора с вариантами;
-   **с изменяемым типом цены:** цена изменится только у тех товаров, у которых в карточке товара разрешено редактирование цены. Для остальных товаров сохранится их текущий тип цены;
-   если выбрать вариант **На все товары**: цена изменится у всех товаров.

## **Вкладка Товары**

![](/mobilnyj-klient/help60.png)

Табличную часть документов можно [настроить](/integraciya-s-1s-8-ut-10/servis/nastrojka-kolonok-dokumentov), изменив отображаемые данные.

Для удобства агента наименование товаров с различными свойствами выделены цветом:

-   Зелёный. Выделены товары, имеющие [план или историю продаж](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha/istoriya-i-plan-prodazh).
-   Фиолетовый. Выделены товары, для которых загружена [спец цена клиента](https://mobi-c.ru/help/mobi_androidclientprice.htm).
-   Синий. Выделены товары из соответствующего Канала сбыта для выбранного Клиента. [Каналы сбыта в 1С 8.х](/integraciya-s-1s-8-ut-10/merchendajzing/formaty-torgovyh-tochek)
-   Оранжевый. Выделены товары, которым передаётся признак цвета из внешней обработки. Например, так можно выделить товары-новинки. Для передачи цвета смотрите раздел [Формат обмена данных](/dlya-razrabotchikov/format-obmena-dannyh/FullLoad).
-   Красный. Выделены товары, у которых количество предполагаемой продажи превышает текущее наличие товара на складе.

 Если товар подпадает под несколько условий, то приоритет цветов идет снизу вверх. Оранжевый главнее синего.

## **Диалог ввода количества (карточка товара)**

При нажатии на строку списка товаров открывается диалог ввода количества.

Примечание: Реквизиты **Бонус**, **Остаток** и **Фейсинг** можно [переименовать или скрыть](/integraciya-s-1s-8-ut-10/servis/nastrojka-kolonok-dokumentov).

![](/mobilnyj-klient/help61.png)

|     |     |
| --- | --- |
| Элемент | Назначение |
| Кол-во | Поле ввода количества реализуемого товара. При смене единицы введённое количество будет пересчитано в новую единицу. **Дробное количество** можно вводить для товаров, у которых установлен признак [Весовой](/voprosy-i-otvety/vvod-drobnogo-kolichestva) в 1С. Для всех остальных дробная часть при сохранении будет сбрасываться. |
| Список выбора единицы | Выбор единицы товара. Список единиц и единица **по умолчанию** задаются в [настройках агента](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/nomenklatura) в обработке Моби-С. |
| Бонус ([отключаемое](/integraciya-s-1s-8-ut-10/servis/nastrojka-kolonok-dokumentov)) | В некоторых торгово-розничных компаниях существует схема работы, когда при выписке определённого количества товара к нему начисляется некоторое количество в виде бонусных бесплатных единиц. Способ предоставления бонусов задается в [настройках агента](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha). |
| Наличие | Остаток товара по выбранному складу, в выбранной единице. Вид отображения остатка можно изменить в [настройках агента](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/sklady). |
| Список выбора склада | Выбор склада. При выборе склада обновляется информация в поле **Наличие**. Склад доступен только, если в учетной системе включен **мультискладской** учет для документов Заказ и Продажа. |
| Остаток ([отключаемое](/integraciya-s-1s-8-ut-10/servis/nastrojka-kolonok-dokumentov)) | Ввод мерчендайзинговой информации об остатке товара в торговой точке. Для просмотра используется отчет [Остатки товаров в торговой точке](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/nomenklatura). Ввод остатка может потребоваться для расчета значения **План**. |
| Фейсинг ([отключаемое](/integraciya-s-1s-8-ut-10/servis/nastrojka-kolonok-dokumentov)) | Ввод мерчендайзинговой информации о фейсинге товара в торговой точке. Для просмотра используется отчет [Остатки товаров в торговой точке](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha). |
| История и План | Отображение [истории и плана продаж](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha/istoriya-i-plan-prodazh). |
| Цена | Текущая цена товара за выбранную единицу. Возможность редактирования цены задается в [настройках агента](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/ceny-i-skidki). |
| Список выбора типа цены | При выборе значений из этого списка будет изменяться реквизит **Цена**. Тип цены предназначен только для изменения цены и не выгружается в 1С. По умолчанию стоит тип цены выбранный на вкладке **Клиент**. Если список активен, значит торговому агенту разрешено изменять цену товара. |
| Скидка | Сумма скидки |
| Список выбора скидки | Выбор скидки. Скидки задаются в [настройках агента](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha). Если запрещено редактирование цены, то список выбора скидки будет тоже заблокирован. |
| Сумма | Сумма проданного товара с учетом скидки. |
| [Дополнительные реквизиты](/integraciya-s-1s-8-ut-10/servis/nastrojka-dop-rekvizitov-na-kpk) | Для каждого документа есть возможность создать персональный набор дополнительных реквизитов, которые будут отображаться в карточке товара. |

Под полями ввода отображается произвольная информация о товаре. Эта информация задается в реквизите **Описание** подзапроса [Goods](/dlya-razrabotchikov/format-obmena-dannyh/FullLoad). Ниже расположены фото товара и кнопка для добавления новых фото с помощью встроенной камеры мобильного устройства.

Примечание: Есть быстрый ввод количества "свайпом" без захода в карточку товара. Нажмите пальцем на строке товара и держите до тех пор, пока не появится иконка со стрелочками. Не отрывая ведите палец вправо или влево. В зависимости от направления будет увеличиваться или уменьшаться количество товара.
