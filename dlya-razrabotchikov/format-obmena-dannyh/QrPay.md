# QrPay - запрос на формирование QR-кода для оплаты

```plaintext
<ВерсияПротоколаОбмена>\r\n
<Платформа>\r\n
<КодАгента>\r\n
QrPay\r\n
[<ТипQR>\t<ИДДокумента>\t<СоздатьНовыйКод>\r\n]
Ответ:
OK\r\n
[<СсылкаНаОплату>\t<tСтатус>\r\n]
\r\n
```

| Параметр | Назначение |
| --- | --- |
| ТипQR | Тип формируемого кода, для СБП - "sbp" |
| ИДДокумента | ИДДокумента в формате выгружаемом для архива документов и возвращаемом на КПК при выгрузке документа с КПК |
| СоздатьНовыйКод | "1" или "0" - если выгружено "1" - то ссылка на оплату перегенерируется. |
| **Ответ** |     |
| СсылкаНаОплату | Возвращаемая на КПК ссылка для оплаты (именно ссылка закодирована в QR коде СБП). |
| Статус | Одно из значений<br><br>DECLINED - Отменена - Операция отменена или истек срок жизни платежной ссылки, для обновления необходимо изменить данные оплаты.<br><br>SUCCESS - Выполнена - Остаток задолженности по документу "Заказ клиента ТД00-000009 от 18.11.2025 11:17:23" равен 0. Документ полностью оплачен.<br><br>IN\_PROGRESS - Выполняется - Ссылка для оплаты сформирована и готова к отправке покупателю. Сумма к оплате ХХ руб. |
