Обработка предназначена для сброса или изменения общих настроек обработки Моби-С. Поддерживаются конфигурации на управляемых формах УТ11, БУ3, УНФ и все построенные на их базе.

  1. Закройте текущую обработку Моби-С.
  2. Скачайте обработку СбросНастроек (УФ).epf
  3. Запустите обработку в вашей 1С через Файл ⇒ Открыть.

РССохраненные настройки имя объекта — префикс настроек, для типовых версий обработки, по умолчанию Mobi-C. Не трогайте если не знаете что это.

Поле ввода Основной каталог — показывает какой каталог назначен основным если обработка Моби-С утеряна и нет уверенности что она типовая. Путь к каталогу можно отредактировать и по нажатию Установить записать настройку в базу, редактирование только вручную чтобы можно было установить любой каталог, по выбору пользователя. Поверка работоспособности Моби-С, доступности каталога и тд не производится.К

Кнопка Отключить UPNP — сетевая инфраструктура может поддерживать проброс порта по технологии UPNP. Если этот функционал использовался, а новое оборудование после обновления не поддерживает данную технологию то запуск обработки может быть заблокирован. Использовать если обработка не стартует или стартует, выводит ошибку UPNP и закрывается.

Настройки Основной каталог  и  Пробросить порт через UPnP находятся в Сервис ⇒ Параметры ⇒ вкладка Модуль интеграции.

Кнопка Отключить режим отладки — отключает флаг режим отладки. В обработке Моби-С, галка Включить режим отладки, включает расширенное логирование обмена. Ставьте галку только если вам нужна информация для поиска ошибок в работе Моби-С.

Поле ввода Версия обработки — позволяет изменить зафиксированную в базе версию обработки Моби-С, применяется при переходах между версиями обработок вниз-вверх для вызова обработчиков обновления. При обычном обновлении обработки использовать не нужно.

Поле ввода Сбросить настройку — позволяет очистить любую настройку любого агента, например для очистки списка супервайзеров без возможности запуска с полными правами (ссылка на супервайзеров агента в УТ11).
