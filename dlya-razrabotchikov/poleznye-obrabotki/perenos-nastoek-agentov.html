Обработка предназначена для переноса настроек агентов из одной базы в другую. Поддерживаются конфигурации на управляемых формах УТ11, БУ3, УНФ и все построенные на их базе.

  1. Скачайте обработку ИмпортЭкспортНастроекАгентов (УФ).epf
  2. Запустите обработку в вашей 1С через Файл ⇒ Открыть.

 

Выгружаются настройки всех агентов имеющихся в базе. Префикс — Mobi-C, изменить его можно через конфигуратор в основной форме обработки.

Выгрузка настроек агентов происходит по следующей схеме:

  1. В базе, из которой выгружаются настройки, формируется список агентов Моби-С.
  2. Для каждого из найденных агентов находятся все имеющиеся настройки.
  3. Каждая найденная настройка сохраняется в Файл для выгрузки.

Особенности загрузки настроек

  1. При загрузке настроек соответствие агентов из базы выгрузки агентам из базы загрузки происходит по коду агента(имя пользователя).
  2. При загрузке не проверяется корректность устанавливаемых отборов.
  3. Если при загрузке настроек не удается найти объект (элемент базы был удален), то значение будет заполнено пустой ссылкой указанного типа.
  4. После выполнения загрузки желательно вручную открыть настройки каждого из агентов и нажать Записать для проверки корректности установленных настроек.
  • Если Вы перенесли данные из одной базы в другую и это конфигурации одного семейства (УТ11-КА2-ERP, БП3, или УНФ3-Розница3), с сохранением идентификаторов объектов, то настройки загрузятся корректно.
  • Если Вы при переходе вы меняете семейство конфигураций, или загружаете НСИ через выгрузку загрузку через Excel – файлы то перенести настройки агентов через эту обработку не получится.
  • В настройках агента используются ссылки на объекты базы данных 1С, корректный перенос настроек возможен в том, и только в том случае если идентификаторы объектов в базе источнике и базе приемнике совпадают.

Для хранения настроек агентов в 1С используется объект ХранилищеОбщихНастроек.

  • КлючОбъекта — либо MobiS для общих настроек, либо код агента для индивидуальных
  • КлючНастроек — префикс (в типовых всегда Mobi-C, ссылка на запуск в одной базе с разными настройками) и наименование настройки
  • Настройки — само значение настройки
  • ОписаниеНастройки — не используется
  • ИмяПользователя — всегда MobiS
