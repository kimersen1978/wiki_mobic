В [Основном каталоге](/voprosy-i-otvety/osnovnoj-katalog-mobi-s) Моби-С находиться файл **mobi\_s.sqlite3**. Фай **mobi\_s.sqlite3** является вспомогательной базой данных в которой хранятся данные нужные для работы функций Моби-С. Формат базы SQLite. Посмотреть содержимое  **mobi\_s.sqlite3** можно программой [DB Browser for SQLite](https://sqlitebrowser.org/).

Таблица **OSTATKI** -  функция [Остатки товаров в торговых точках](/integraciya-s-1s-8-ut-10/merchendajzing/ostatki-tovarov-v-torgovyh-tochkah).

Таблица **messages** \- функция [Сообщения](/integraciya-s-1s-8-ut-10/agenty/soobshcheniya).

Таблица **Log** \- функция [Лог действий агентов](/integraciya-s-1s-8-ut-10/agenty/log-dejstvij-agentov). 

Таблица **PhotoMetadata** - журнал [Фотографии](/integraciya-s-1s-8-ut-10/merchendajzing/fotografii), информацию из метаданных файлов фотографий. Данные попадают в таблицу после формирования списка фотографий в журнале **Фотографии**.

Таблица **IncomingPhotos** - журнал [Фотографии](/integraciya-s-1s-8-ut-10/merchendajzing/fotografii),  информацию по привязке файла фотографии к объекту сьемки. Данные попадают в таблицу при выполнении запроса на загрузку фотографий из КПК в 1С.

## Ошибка disk image is malformed

При открытии обработки или в процессе работы в логе появляются сообщения: **database disk image is malformed.** Поврежден файл базы данных **mobi\_s.sqlite3.** Причиной ошибок в базе данных sqlite может служить повреждение данных в файле (физические/логические повреждения жесткого диска, последствия вирусов, ошибки при копировании файла).

### Вариант 1. Удаление поврежденной баз

Если в базе **mobi\_s.sqlite3** для вас нет ничего ценного то её можно просто удалить.

1.  Закрыть все обработки Моби-С и саму базу 1С.
2.  Перейти в [Основной каталог](/voprosy-i-otvety/osnovnoj-katalog-mobi-s) Моби-С и удалить файл **mobi\_s.sqlite3.**
3.  Запустить обработку Моби-С. Будет создан новый чистый файл файл **mobi\_s.sqlite3.**  
     

### **Вариант 2. Восстановление поврежденной базы**

Восстановление поврежденной базы данных производится с помощью утилиты командной строки sqlite.

[http://www.sqlite.org/download.html](http://www.sqlite.org/download.html), необходимо скачать «sqlite-tools-win32-x86...zip», в описании которой указано **including the command-line shell**, на текущий момент актуальная ссылка

[https://www.sqlite.org/2022/sqlite-tools-win32-x86-3370200.zip](https://www.sqlite.org/2022/sqlite-tools-win32-x86-3370200.zip)  
 

#### Процедура восстановления

1.  Закройте все работающие обработки Моби-С.
2.  Сделайте резервную копию файла **mobi\_s.sqlite3**.
3.  Скопируйте испорченный файл **mobi\_s.sqlite3** в отдельный каталог, например **C:\\1**, далее все операции выполняются в этом каталоге если не указано иное.
4.  В этот каталог необходимо распаковать файл **sqlite3.exe** из архива **sqlite-tools-win32-x86-...zip**.
5.  В этот каталог скачайте 2 файла [ИзвлечьДанныеИзБазы.bat](https://mobi-c.ru/arc/%D0%98%D0%B7%D0%B2%D0%BB%D0%B5%D1%87%D1%8C%D0%94%D0%B0%D0%BD%D0%BD%D1%8B%D0%B5%D0%98%D0%B7%D0%91%D0%B0%D0%B7%D1%8B.bat) и [ПоместитьДанныеВБазу.bat](https://mobi-c.ru/arc/%D0%9F%D0%BE%D0%BC%D0%B5%D1%81%D1%82%D0%B8%D1%82%D1%8C%D0%94%D0%B0%D0%BD%D0%BD%D1%8B%D0%B5%D0%92%D0%91%D0%B0%D0%B7%D1%83.bat).
6.  Выполните файл **ИзвлечьДанныеИзБазы.bat**, выполнение может занять продолжительное время, в результате будет сформирован файл **mobi\_s.sql**.
7.  Текстовым редактором (желательно использовать Notepad++ или AkelPad, нежелательно – Блокнот) откройте файл **mobi\_s.sql**.
8.  Если последняя строка в файле **mobi\_s.sql** начинается с **ROLLBACK**, то замените всю строку на **COMMIT**; и сохраните файл.
9.  В этом каталоге переименуйте файл **mobi\_s.sqlite3** в **mobi\_s\_исходный.sqlite3**.
10.  Запустите файл **ПоместитьДанныеВБазу. bat**, выполнение может занять продолжительное время, могут выводится сообщения об ошибках, игнорируйте их, в результате будет сформирован файл **mobi\_s.sqlite3**.
11.  Скопируйте полученный файл **mobi\_s.sqlite3** в [основной каталог Моби-С](https://mobi-c.ru/help/mobi_filedescr.htm) с заменой существующего и проверьте работу с внешней базой из одной обработки – сообщения, фотографии, остатки в торговых точках. Если ошибок работы с внешней базой данных нет, запускайте остальные обработки.
