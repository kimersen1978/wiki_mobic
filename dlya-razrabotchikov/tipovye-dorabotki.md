# Интеграция Моби-С с 1С УТ11.epf

### Изменить период и количество документов для выгрузки истории продаж

Стандартные параметры выгрузки [истории продаж](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha/istoriya-i-plan-prodazh)  2 месяца и 3 документа

### Интеграция Моби-С с 1С УТ11.epf, Интеграция Моби-С с 1С УНФ.epf

Метод: **ОбработкаМобиС.ФормаОсновная.ПодготовитьДанныеSalesHelp**

**КолвоМесяцевНазадДляИстории** - период просмотра документов для формирования истории продаж  
**КолвоМесяцевНазадДляПлана** - период просмотра документов для формирования плана продаж  
  
**МаксКолвоПосещенийПродажа** - максимальное количество документов **Продажа** в истории  
**МаксКолвоПосещенийВозврат** - максимальное количество документов **Возврат** в истории  
**МаксКолвоПродажаВозврат** - максимальное количество документов (**Продажа + Возврат**) в истории

### Дополнение документа многооборотной тарой если отключено использование соглашений

Метод: **ОбработкаМобиС.ФормаОсновная.**ЗаполнитьРеквизитыЗаказа

Метод: **ОбработкаМобиС.ФормаОсновная.**ЗаполнитьРеквизитыНакладной

в ветке "**Иначе**" условия "**Если ИспользоватьСоглашения Тогда**" необходимо раскомментировать условие "**Если ЕстьМногооборотнаяТара() Тогда**"

### Корректировка документа при отображении долга по документу на КПК

Доработка позволяет учесть корректировку документа при отображении долга по документу на КПК

**ОбработкаМобиС.ФормаОсновная.ПодготовитьМассивАрхивДокументовРоссия**

// для учета корректировок раскомментрируйте следующую строку  
//ДокументыСДолгамиПоСрокамДокумент = "ОбъектРасчетов.Объект";  
  
Сумма документа в архиве на КПК не изменится, а долг по документу отобразится с учетом корректировки.  
при работе с корректировкой реализации в УТ11 есть следующие моменты

1.  Отчет по долгам из справки Моби-С может показывает долги БЕЗ учета корректировки, следовательно при внесении данной правки опираться на этот отчет для проверки данных нельзя
2.  Подбор оснований для ПКО показывает долг по документу С учетом корректировки

### Исключение Заказа / Продажи из архива документов если Продажа создана на основании Заказа

По умолчанию в Архив документов выгружаются все документы, согласно настроек в настройках агента.  
Если на основании **Заказа** создана **Продажа** \- то на КПК будут выгружены оба документа.  
  
Если необходимо исключить "дублирование" документов то необходимо внести изменения в код обработки в методе ОбработкаМобиС.ФормаОсновная.ПодготовитьМассивАрхивДокументовРоссия()  
  
Необходимо включить режим **ВыгружатьТолькоОдинДокументЕслиЕстьЗаказИПродажа**  
  
было  
**ВыгружатьТолькоОдинДокументЕслиЕстьЗаказИПродажа = Ложь;**  
стало  
**ВыгружатьТолькоОдинДокументЕслиЕстьЗаказИПродажа = Истина ;**  
  
и выбрать какой документ будет выгружать при наличии "дубля"  
для выгрузки **Заказа** необходимо  
**ВыгружатьПродажуЕслиЕстьЗаказИПродажа = Ложь;**  
  
для выгрузки **Продажи** необходимо  
**ВыгружатьПродажуЕслиЕстьЗаказИПродажа = Истина;**  
 

// добавляем возможность выгрузки всех проведенных заказов клиента независимо от их статуса

**ВыгружатьВсеЗаказы** \= **Истина**;

  
доработка актуальная для версии **5.5.74** или выше

### Изменение механизма выгрузки неуточненных цен в режиме выгрузки уточненных цен

Стандартное поведение, на неуточненные товары выгружается цена с типом **Тип цен по умолчанию**.  

После доработки для неуточненных товаров выгружается цена по типу цен из основного договора контрагента, если он не задан - то тип цен по умолчанию.

Метод: **ОбработкаМобиС.ФормаОсновная.ВыгрузитьЦены()**

в это методе нужно найти комментарий "// условие формирования уточняющих цен" и на строку выше изменить "?(Истина" на "?(Ложь"

# Все обработки

Вернуть старый алгоритм заполнения комментария в документах. В старых обработках в комментарий писалась информация **КодДокумента+ДатаВремяДоставки+Комментарий**, в новых обработках **КодДокумента** исключен.

**ОбработкаМобиС.ФормаОсновная** делаем поиск по подстроке **Док.Комментарий.** Смотрим название процедуры, если оно **ЗаполнитьРеквизитыЗаказа** и **ЗаполнитьРеквизитыНакладной** и пр**.**. 

Раскомментирован верхний блок кода, закомментировать нижний блок кода заполнения комментария.
