В запросе **FullLoad** есть подзапрос **Properties**. В этом разделе мы расскажем как его можно использовать.

Подзапрос Propertis выгружает:

-   [Каналы сбыта(ассортиментные или товарные матрицы)](/integraciya-s-1s-8-ut-10/merchendajzing/formaty-torgovyh-tochek)
-   Категории для документов
-   [Переменные и их значения для анкет](/funkcii/redaktor-form/rabota-s-peremennymi)
-   Дополнительные свойства

Properties\\r\\n  
\[<Свойство>\\t<КодТаблицы>\\t<КодОбьектаТаблицы>\\t<ЗначениеСвойства>\\r\\n\]

## **Выгрузка каналов сбыта**

Для понимания этого раздела вам необходимо ознакомится с теоретической [информацией по созданию и работе с каналами сбыта](/integraciya-s-1s-8-ut-10/merchendajzing/formaty-torgovyh-tochek).

| Параметр | Назначение |
| --- | --- |
| Свойство | Строка "КаналСбыта". |
| КодТаблицы | Тип объекта 1 - клиент, 4 - товар. |
| КодОбъектаТаблицы | Код товара или клиента. |
| ЗначениеСвойства | Наименование канала сбыта. |

### Пример: Выгрузка каналов сбыта

Есть клиент "Алхимов" с кодом "к001".

Есть товар "Товар 1" с кодом "т001", "Товар 2" с кодом "т002", "Товар 3" с кодом "т003".

"Товар 1" и "Товар 3" входит в товарную матрицу для клиента "Алхимов".

`Ответ=Ответ+"КаналСбыта"+Таб+"1"+Таб+"к001"+Таб+"Алхимов"+РазделительСтрок;`  
`Ответ=Ответ+"КаналСбыта"+Таб+"4"+Таб+"т001"+Таб+"Алхимов"+РазделительСтрок;`  
`Ответ=Ответ+"КаналСбыта"+Таб+"4"+Таб+"т003"+Таб+"Алхимов"+РазделительСтрок;`

После загрузки этих данный на КПК торгового агента при выборе клиента "Алхимов", в любом документе с табличной частью, товары входящие в его товарную матрицу будут выделены синим цветом. Так же можно установить фильтр, что бы отобрать только товары из товарной матрицы клиента.

## **Выгрузка категорий документов**

Категории это дополнительный параметр который можно выбрать в документе на КПК.

|     |     |     |
| --- | --- | --- |
| Параметр | Тип в КПК | Назначение |
| ∗Свойство | строка | Уникальный код категории. |
| ∗КодТаблицы | int | Внутренний код документа на КПК.<br><br>12 – Продажа, 14 – Заказ, 16 – Покупка, 18 - ПКО   19 - Анкета |
| ∗КодОбьектаТаблицы | строка | пусто |
| ЗначениеСвойства | строка | Отображаемое наименование категории. |

### Пример: Добавим к документу Продажа категорию "Оплата наличными".

Для 1С 7.7 строка добавления категории будет выглядеть так. Для 1С 8 смотрите пример в коде модуля интеграции Моби-С.

`Ответ=Ответ+1+Таб+12+Таб+""+Таб+"Оплата наличными"+РазделительСтрок;`

### **Сохраняем в 1С результаты выбора категорий документа на КПК**

В протоколе обмена при выгрузке любого документа(Invoice, Request, ReceiptInvoice, PKO, Anketa, Inventory) выгружаются коды выбранных категорий.

\[<КодКатегории>\\t\]

В модуле интеграции Моби-С для 1С 8 сохранение категорий реализовано в полном объеме. Для 1С 7 будет зависеть от того как в конфигурации реализована работа с категориями. При выгрузке у вас на руках есть информацию о документе и кодах выбранных категорий.

## **Выгрузка переменных и их значений для анкет**

Для понимания этого раздела вам необходимо ознакомится с теоретической [информацией по созданию и работе с переменными в анкетах](/funkcii/redaktor-form/rabota-s-peremennymi).

|     |     |     |
| --- | --- | --- |
| Параметр | Тип в КПК | Назначение |
| ∗Свойство | строка | Название переменной |
| ∗КодТаблицы | int | "1" |
| ∗КодОбьектаТаблицы | строка | Код клиента. |
| ЗначениеСвойства | строка | Значение переменной. |

### Пример: Добавим к стандартной анкете Мерчендайзинг значение переменной SKU

Мы хотим, что бы для клиента "Алхимов" с кодом "К001" в анкетах для переменной "SKU" отображалось значение 12.

`Ответ=Ответ+"SKU"+Таб+"1"+Таб+"К001"+Таб+"12"+РазделительСтрок;`

В результате на КПК при выборе агента "Алхимов" во всех анкетах где используется переменная SKU будет отображаться цифра 12. Для всех остальных клиентов значение SKU будет пустым.

## **Выгрузка информации о минимальной сумме заказа**

Выгрузка информации для контроля минимальной суммы в документах Заказ и Продажа. При включении контроля документы, созданные на меньшую сумму, не могут быть выгружены.

|     |     |     |
| --- | --- | --- |
| Параметр | Тип в КПК | Назначение |
| ∗Свойство | строка | Строка "МинСуммаРеализации" или "МинСуммаЗаказа" |
| ∗КодТаблицы | int | "1" |
| ∗КодОбьектаТаблицы | строка | Код клиента. |
| ЗначениеСвойства | строка | Минимальная сумма заказа для данного клиента. |

Есть клиент "Алхимов" с кодом "к001".

Зададим минимальную сумму для документа Продажа в 1000 рублей.

`Ответ=Ответ+"МинСуммаРеализации"+Таб+"1"+Таб+"к001"+Таб+"1000"+РазделительСтрок;`

Зададим минимальную сумму для документа Заказ в 1500 рублей.

`Ответ=Ответ+"МинСуммаЗаказа"+Таб+"1"+Таб+"к001"+Таб+"1500"+РазделительСтрок;`

## **Выгрузка информации о минимальном количестве товара**

Выгрузка информации для контроля минимального количества в документах Заказ и Продажа. При включении контроля не возможно добавить количество товара к продаже меньше минимального.

|     |     |     |
| --- | --- | --- |
| Параметр | Тип в КПК | Назначение |
| ∗Свойство | строка | Строка "МинЕдПродажи". |
| ∗КодТаблицы | int | "4" |
| ∗КодОбьектаТаблицы | строка | Код товара. |
| ЗначениеСвойства | строка | Коэффициент минимальной единицы |

Есть товар "Сок яблочный" с кодом "т001".

Зададим минимальную количество к продаже для документов Заказ и Продажа 4 базовые единицы.

`Ответ=Ответ+"МинЕдПродажи"+Таб+"4"+Таб+"т001"+Таб+"4"+РазделительСтрок;`

Все данные выгруженные через этот запрос и привязанные к товарам отображаются в качестве параметров фильтра в документах **Заказ** и **Продажа** на вкладке [**Свойства**](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha/poisk-tovara).
