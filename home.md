# Мобильная торговля Моби‑С

**Разработчик:** ИП Тропынин Игнатий Олегович
**Сайт:** [mobi-c.ru](https://mobi-c.ru/requisite.html)  
**Свидетельство о государственной регистрации программ для ЭВМ:** № 2011616483
**Регистрация в реестре российского ПО:** [Реестровая запись №1066 от 01.06.2016](https://reestr.digital.gov.ru/reestr/302415/?sphrase_id=17316891)

Программный комплекс «Моби‑С» предназначен для автоматизации мобильной торговли и состоит из двух основных частей: серверной и мобильной.

* **Мобильная часть** работает на карманных персональных компьютерах (КПК) и предназначена для использования торговыми представителями.
* **[Сервер](/ru/requirements-server)** работает на сервере или обычном ПК и состоит из двух подсистем:
    * PocketServer.dll;
    * Модуль интеграции.

## Мобильная часть

Мобильная часть представляет собой программу, устанавливаемую на КПК или коммуникатор с Google Android.

Установленная и настроенная программа даёт возможность мобильному агенту работать с локальной учётной системой. Программа позволяет:

* загружать актуальную для агента информацию из учётной системы;
* выгружать созданные документы и прочие данные.

## Учётная система с запущенным модулем интеграции

Внешний отчёт — основной элемент взаимодействия торгового агента с учётной системой.

Внешний отчёт отвечает за:

* состав выгружаемых на КПК данных;
* формирование и учёт созданных в КПК документов.

Внешний отчёт открывается в клиенте 1С. Поддерживаются платформы:

* 1С:Предприятие 8.3;
* 1С:Предприятие 8.2;
* 1С:Предприятие 7.7.

Для каждой платформы и конфигурации 1С используется [свой модуль интеграции](https://mobi-c.ru/1cconfiguration.html).

## Библиотека PocketServer.dll

Сервисный компонент — является связующим звеном между учётной системой и мобильной частью.

После получения запроса от мобильной части данные запроса передаются во внешний отчёт, который в зависимости от типа запроса:

* либо принимает данные в учётной системе;
* либо подготавливает данные для выгрузки в КПК.

[PocketServer.dll](/dlya-razrabotchikov/pocketserver-dll) разработана по технологии создания внешних компонент 1С.


## Протокол обмена

Обмен данными производится на базе протокола обмена.

Особенности протокола:

* все запросы и ответы представляют собой текстовые строки, разделяемые символами CR/LF (за исключением ответа на запрос на обновление программы);
* весь поток данных от сервера к КПК сжимается по алгоритму Zip;
* работа по сжатию выполняется в PocketServer.dll.
