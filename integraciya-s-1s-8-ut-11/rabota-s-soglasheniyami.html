Формирование списка соглашений доступных агенту на КПК

Для того чтобы соглашение было доступно торговому агенту на КПК должны выполняться следующие условия:

  1. Использование соглашений должно быть включено: Администрирование ⇒ Продажи ⇒ Оптовые продажи ⇒ Использование соглашений с клиентами.
  2. Соглашение должно удовлетворять используемым в (1). Индивидуальным считается соглашение в котором указано с каким именно клиентом оно заключено (поле Клиент), при этом поля Контрагент и Организация могут быть не заполнены. Типовыми считаются все остальные соглашения.
  3. Соглашение должно быть действующим: статус соглашения Действует.
  4. Соглашение должно действовать на дату выгрузки в КПК. Период действия соглашения должен включать дату выгрузки, если верхняя/нижняя граница периода действия не задана, то считается что период сверху/снизу неограничен.
  5. Если в соглашении указан Склад, то данный склад должен быть в списке доступных агенту складов.[1]
  6. Если в соглашении указан Вид цены, то данный вид цены должен быть в списке доступных агенту типов цен. Если в настройках агента выключена настройка Типы цен из соглашений, то на КПК будут выгружены только соглашения с типами цен, указанными в списке разрешенных типов цен, иначе будут выгружены все соглашения и список типов цен на КПК будет сформирован из выгруженных соглашений.
  7. Если в соглашении указана организация, то данная организация должна быть доступна в списке доступных агенту организаций.
  8. Если в соглашении не указана организация, то при выгрузке на КПК соглашение будет выгружено отдельно для каждой доступной агенту организации, название организации будет добавлено к названию соглашения.
  9. Соглашение должно быть доступно для клиента выгруженного на КПК, если клиент на КПК не выгружаются - соглашение выгружено не будет.
  10. Если в настройках агента на вкладке Клиенты установлен признак Выгружать на кпк только используемые в работе торговых представителей соглашения, то в соглашении должен быть установлен признак Используется в работе торговых представителей. Если данной настройки в соглашении нет, то необходимо её включить: Администрирование ⇒ Продажи ⇒ Торговые представители ⇒ Ведется работа через торговых представителей.[2]
  11. По умолчанию выключено использование соглашений предков.[1]
  12. Договора к соглашениям на КПК не выгружаются! Подстановка договора осуществляется в 1С в момент формирования выгруженного с КПК документа. Выбирается первый договор, совпадающий с выбранным соглашениям по реквизитам: Партнер (с учетом Родителей(иерархии)), Контрагент, Организация, Хозяйственная операция, Валюта взаиморасчетов, Порядок расчета по договорам. Договор должен быть действующим и должен действовать на дату выгрузки документа (см 4).

[1] включение/отключение через конфигуратор

[2] настройка применяется когда агентам доступны не все существующие с клиентом соглашения

Если в настройках агента указано Получать V клиентов и V соглашения из Условия обслуживания партнеров торговыми представителями то список соглашений берется с учетом Условия обслуживания партнеров торговыми представителями. Сначала выбираются доступные клиентам соглашения, затем они ограничиваются соглашениями из Условия обслуживания партнеров торговыми представителями т. е. если соглашение недоступно агенту (закрыто, вышел срок действия, недоступно по типу цены или складу…), но указано в Условия обслуживания партнеров торговыми представителями, то на КПК оно выгружено НЕ БУДЕТ.

Работа с соглашениями при выгрузке документа из КПК в 1С

Если в конфигурации запрещена работа с соглашениями (см 1.Использование соглашений…) то соглашение не заполняется: организация берется из шапки документа с КПК, Валютой и Валютой взаиморасчетов документа устанавливается Валюта регламентированного учета системы, налогообложение берется на основании организации и склада на дату документа.

Если в работа с соглашениями разрешена тогда:

  1. Если на КПК выбрано соглашение (поле Договор на КПК) тогда соглашением является выбранное агентом соглашение.
  2. Если в соглашении указана организация, то организация берется из соглашения, иначе из шапки документа на КПК.
  3. Если в соглашении указан склад, то складом документа становится склад из соглашения, иначе если на КПК выбран склад то он становится складом по документу, иначе берется первый склад из настроек агента.[3]
  4. Валюта, ВалютаВзаиморасчетов, НалогообложениеНДС и ЦенаВключаетНДС - берется из соглашения.

[3] если включен мультискладской учет, то берется склад/группа складов включающая все склады из табличной части и разрешенная к использованию в документах.

Соглашения для поставщиков - по аналогичной схеме с точностью до наличия/отсутствия реквизитов.

Подбор соглашения в документ если с КПК выгружено «Авто»

Выбираются в порядке указанном в настройке Сортировка соглашений Эта настройка позволяет задать порядок выбора соглашений на КПК. Сортировка применяется отдельно к индивидуальным и типовым соглашениям. Если для клиента есть индивидуальное соглашение - оно всегда будет выше любого типового.

Значение Сортировка по умолчанию, фиксированного порядка в  индивидуальных и типовых соглашениях нет, по умолчанию будет выбрано первое соглашение из списка доступных.

Индивидуальные цены из соглашения

В соглашении можно указать уточнение цен по ценовым группам и по товарам при продаже/заказе через это соглашение. Уточненные цены выгружаются на КПК при выполнении с КПК запроса Запросить цены контрагента, приоритет выгрузки цен следующий - первыми выгружаются уточняющие цены для товаров, затем выгружаются уточняющие цены по ценовым группам. Если цена товара уточняется напрямую и через ценовую группу, то будет выгружено только уточнение цены для товары.

Коррекция

Коррекция может выполняться только по соглашению с заполненным реквизитом Вид цены.
