На этой закладке задаются условия отбора контрагентов, выгружаемых в КПК.

![](/help52.png)

Флажки **Получать Клиентов или Соглашения из** - если флажки установлены, то Клиенты/Соглашения выгружаются на КПК согласно **Условий обслуживания партнеров торговыми представителями**. Прочие установленные отборы при этом игнорируются.

## **Отбор клиентов**

Для формирования списка клиентов (пара Контрагент+Партнер), выгружаемых на КПК можно использовать любой из отборов и любые их комбинации.

-   По Контрагентам
-   По Партнерам
-   По Сегментам партнеров

При настройке отбора **По Контрагентам** или **По Партнерам** выгружаемый элемент справочника должен присутствовать в выборке из справочника отбором.

При настройке отбора **По Сегментам партнеров** используются следующие правила:

-   Если у партнера, включенного в сегмент, есть контрагенты, то на КПК будут выгружены все подчиненные партнеры данного партнера  
    **Пример**:  Сегмент 'Торговые сети города', в него входят головные партнеры 'Сеть1', 'Сеть2' и 'Сеть3', каждому соответствует контрагент 'Сеть1', 'Сеть2' и 'Сеть3'. Конкретные ТТ каждой сети называются 'Сеть1\_1', 'Сеть2\_1', 'Сеть2\_2', 'Сеть3\_1', 'Сеть3\_3'. Тогда при настройке фильтра Сегмент = 'Торговые сети города', на КПК будут выгружены головные партнеры, и все подчиненные ТТ.
-   Если у партнера, включенного в сегмент, нет контрагентов, и у партнера родителя есть контрагенты, то на КПК будет выгружен только рассматриваемый партнер, без подчиненных.  
    **Пример**: Те же условия что и выше, '\_1' - район, где расположена ТТ в сегмент 'район\_1' входят 'Сеть1\_1', 'Сеть2\_1' и 'Сеть3\_1'. Тогда при настройке фильтра Сегмент = 'район\_1', на КПК будут выгружены только указанные ТТ.

Если список клиентов доступных агенту формируется на основании **Условия обслуживания клиентов торговыми представителями**, то действуют те же правила отбора, что и для отбора по списку партнеров. Признак "**Только по отбору**" используется в том случае, если на КПК необходимо выгрузить только клиентов указанных в отборе, без дополнения потомками.

**Пример**: Партнер П1. Партнеры П1\_1, П1\_2 и П1\_3 - подпартнеры П1. Контрагент К1, партнером контрагента указан П1. Отбор настроен следующим образом - агенту доступен только партнер П1. Если "Только по отбору" включено, то на КПК будет только П1(К1). Если "Только по отбору" выключен, то на КПК будут: П1(К1), П1\_1(К1), П1\_2(К1) и П1\_3(К1).

На список выгружаемых клиентов оказывает влияние настройка

**Администрирование ⇒ Маркетинг и CRM ⇒ Настройки CRM ⇒ Независимо вести партнеров и контрагентов**, если настройка не активирована, то в выгрузку попадают только партнеры и привязанные к ним контрагенты, без учета контрагентов родителей.

Нужно иметь в виду, что несколько строк фильтра соединяются логическим **И** т.е. контрагент должен удовлетворять каждой строке фильтра.

> После того как установите фильтры проверить результат можно нажав кнопку **Клиенты и соглашения агента.** Отобразиться список клиентов и соглашений которые будут выгружены на КПК агента.

Кнопка **Клиенты агента на карте** открывает диалог [**Координаты клиентов**](/integraciya-s-1s-8-ut-11/planirovanie-i-kontrol/planirovanie-i-kontrol). Отображает на онлайн карте всех клиентов с GPS координатами.

Кнопка **Остатки у клиентов** формирует [отчет](https://wiki.mobi-c.ru/ru/integraciya-s-1s-8-ut-11/merchendajzing/ostatki-tovarov-v-torgovyh-tochkah), который предназначен для просмотра информации по собранным остаткам и фейсингу товара в торговых точках.

Кнопка **Выделить клиентов цветом** - открывает диалоговое окно **Настройка цветов сегментов,** в котором можно задать выделение цветом контрагентов на основании сегментов контрагентов.

Раздел **Клиент по умолчанию** позволяет задать клиента, который будет подставляться при создании нового документа на КПК,

Поле **Скрыть сегмент партнеров по дополнительному сведению** - скрывает сегмент партнеров на КПК при установке определенного свойства, аналогично сокрытию сегмента номенклатуры по доп. свойству.

Флажок **Выгружать на кпк только используемые в работе торговых представителей соглашения** - если установлена галка, то на кпк выгружаются только те соглашения, у которых стоит признак "**используется в работе торговых представителей"**.

Список "**Сортировка соглашений**" позволяет задать порядок выбора соглашений на КПК. Информация [Работа с соглашениями](/integraciya-s-1s-8-ut-11/rabota-s-soglasheniyami). Если для клиента есть индивидуальное соглашение - оно всегда будет выше любого типового.

-   Сортировка по умолчанию - фиксированного порядка в  индивидуальных и типовых соглашениях нет, по умолчанию будет выбрано первое соглашение из списка доступных.
-   По наименованию - алфавитная сортировка по полю **Наименование** соглашения.
-   По номеру - алфавитная сортировка по полю **Номер** соглашения.
-   По "Период действия с" по возрастанию (от старых к новым)
-   По "Период действия с" по убыванию (от новых к старым)
-   По дате создания по возрастанию
-   По дате создания по убыванию
