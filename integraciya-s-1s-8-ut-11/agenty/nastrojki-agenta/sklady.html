На вкладке Склады указываются доступные склады и настраиваются параметры работы с товарными остатками.

Посмотреть список выгружаемых товаров с остатками можно на вкладке Номенклатур», нажав кнопку Показать номенклатуру, выгружаемую на КПК.

На КПК будет выгружен список выбранных складов. 

  • Для добавления склада в список нажмите кнопку Добавить в верхнем тулбаре.
  • Для удаления склада из списка, выделите строку и нажмите на клавиатуре кнопку Delete.
  • Кнопки изменения порядка складов. Задают порядок в котором при определенных условиях товару будут списываться со складов при создании документа в 1С. Подробно читайте в разделе Заполнение складов.
  • Кнопка Основной позволяет задать склад по умолчанию. Основной склад выделен в списке жирным шрифтом. При создании документа на КПК будет подставляться склад по умолчанию.

Поле Остаток позволяет задать варианты отображения остатка для выбранного склада на КПК.

  • Отображать - отображается реальное количество остатка товара в выбранной единице.
  • Ограничить остаток - задается максимальное значение. Если товара меньше, то отображается реальное количество, если больше то оно будет ограничено максимальным значением.
  • Скрыть остаток - при наличии товара на складе будет отображаться надпись Есть.
  • Для товаров с типом Услуга, Набор всегда отображается надпись Есть.

Раздел Выгружаемый на КПК Остаток

 

Вариант Остаток по складу

Для того что бы посмотреть текущие остатки по товару в вашей 1С, открываем в карточке товара отчет Остатки и доступность.

Главное ⇒ Номенклатура ⇒ Карточка товара ⇒ Отчеты ⇒ Остатки и доступность

После того как сформировали отчет, выполните на КПК запрос Полная загрузка и сравните остаток в КПК и 1С.

  • По умолчанию выгружаются остатки из столбца Сейчас/Доступно плюс столбец отчета по остаткам Сейчас/В резерве.
  • Включение Сейчас (В резерве) вычитается количество из столбца отчета по остаткам Сейчас/В резерве.
  • Включение Ожидается (Поступит) добавляет количество из столбца отчета по остаткам Ожидается/Поступит.
  • Включение Ожидается (В резерве) вычитается количество из столбца отчета по остаткам Ожидается/В резерве.

 

Вариант Остаток по организации и складу

Остатки на КПК будут разные для каждой организации.

Остатки смотреть отчетом Функции для технического специалиста ⇒ Ведомость по товарам организаций.

 

Флажок Отображать остаток товара на КПК как в 1С - выбора варианта отображения остатка товара на КПК для дробного остатка.

  • Если галка включена – остаток показывается в виде десятичного числа. Пример: 12 блоков по 15 пачек. Продали 3 пачки → остаток 11,8 блоков.
  • Если галка выключена – остаток показывается в виде дроби: целые единицы + остаток в базовых единицах. Пример: 12 блоков по 15 пачек. Продали 3 пачки → остаток 11/12 (11 блоков и 12 пачек).

 

Варианты работы со складами в 1С

Настройка в 1С НСИ и Администрирование ⇒ Продажи ⇒ Оптовые продажи, галка Реализация товаров с нескольких складов. Есть два варианта работы со складами на КПК.

Склад в шапке документа

Настройка Реализация товаров с нескольких складов отключена. На КПК склад выбирается на вкладке Клиент и един для всего документа, выбор склада в карточке товара заблокирован.

 

Склад в карточке товара

Настройка Реализация товаров с нескольких складов включена. В этом случае в документах 1С склад можно указать в шапке, а так же для каждой строки табличной части документа в 1С. В Моби-С на КПК склад выбирается в карточке товара и может быть разным для каждого товара, склада на вкладке Клиент склада нет.

 

Отображение остатка в табличной части документов на КПК

Функция Настройка колонок документов, позволяет получить различные варианты отображения остатка в табличной части документа на КПК.

  • Ячейка <Наличие> - отображается остаток товара по выбранному складу.
  • Ячейка <НаличиеВсеСкл> - отображается  остаток товара сумма по всем складам.
  • Ячейка <Склад> - отображается  наименование выбранного склада.
  • Ячейка <Наличие на основном складе> - отображается  остаток товара на основном складе.
  • Ячейка <Основной склад> - отображается наименование основного склада.

 

Далее описаны алгоритмы по которым заполняются параметры склада для различных документов 1С при выгрузке документов с КПК.

Заполнение складов в документе Реализация товаров и услуг или Заказ покупателя

Настройка в 1С НСИ и Администрирование ⇒ Продажи ⇒ Оптовые продажи, галка Реализация товаров с нескольких складов. Есть два варианта работы со складами на КПК.

 

Настройка “Реализация товаров с нескольких складов” ВКЛЮЧЕНА

Заполнение склада в шапке документа

  • Если в табличной части документа для всех строк использован один склад, то данный склад и будет складом по документу.
  • Если в табличной части документа выбрано несколько складов и они входят в одну группу, то данная группа и будет складом по документу.
  • Если в табличной части документа выбрано несколько складов и они входят в разные группы, то складом по документу будет взят склад из первой строки табличной части документа.

Заполнение склада в табличной части документа

  • Если в карточке товара указан конкретный склад, то данный склад и будет складом в строке документа,  независимо от остатков на этом складе.
  • Если в карточке товара указан склад Авто, товар будет списываться со складов с порядке, в котором они перечислены в списке в настройках агента. Если товара на всех складах недостаточно, то недостающее количество будет списано с первого в списке склада в настройках агента.
     

Настройка “Реализация товаров с нескольких складов” ВЫКЛЮЧЕНА 

Заполнение склада в шапке и табличной части документа

Если используются соглашения, то приоритет выбора складов в документ следующий (по убыванию)

  1. склад из соглашения
  2. склад указанный агентом на КПК
  3. первый склад из настроек агента

иначе приоритет выбора складов в документ следующий (по убыванию)

  1. склад указанный агентом на КПК
  2. первый склад из настроек агента.

Приоритет выбора складов в документ следующий (по убыванию)

  1. склад указанный Агентом на КПК

Резервирование товара в Заказе и расчет способа обеспечения

Информация актуальна только для проведенных документов Заказ.

Настройка в 1С влияющая на обеспечение НСИ и администрирование ⇒ Настройка НСИ и разделов ⇒ Продажи ⇒ Оптовые продажи ⇒ Использование заказов

Флажок Заказ как счет

  • Устанавливается вариант обеспечения Требуется/К обеспечению*, без учета наличия товара на момент заказа или флажка Резервировать.

Флажок Заказ только со склада  или Заказ со склада и под заказ

Проверяется наличие товара на складе (если мультискладской учет то с учетом каждого склада(смотрите информацию выше)). 

При создании заказа на КПК в Моби-С на вкладке Прочее есть поле Резервировать на складе которое может принимать значение Да/Нет.

  • Товар есть на складе - если агент установил на КПК параметр Резервировать на складе в ДА устанавливается вариант обеспечения Со склада\Отгрузить*, иначе вариант Отгрузить.
  • Товара нет на складе - в любом случае устанавливается вариант обеспечения Требуется\К обеспечению.

Если документ не проводной, то обеспечение выбирается по следующей логике - по алгоритмам 1С устанавливается вариант обеспечения "по умолчанию". Значение по умолчанию рассчитывается 1С на основании номенклатуры в текущей строке и реквизитов документа, и может различаться для разной номенклатуры в рамках одного документа.
