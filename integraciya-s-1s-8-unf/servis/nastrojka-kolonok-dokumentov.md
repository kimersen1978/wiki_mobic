Диалог Настройка документа предназначен для настройки табличной части и карточки товара на КПК агента. Табличную часть каждого документа можно настроить "под себя". Для документов [Заказ и Продажа](https://mobi-c.ru/help/mobi_androidrequest.htm) настроить заголовок и скрытие полей **Бонус** **Фейсинг** и **Остаток**.

## **Настройка табличной части**

![](/ut10/ut10-9.png)

Табличная часть состоит из ячеек 2 типов:

-   Ячейка с данными - позволяет выбрать тип отображаемых данных.
-   Ячейка заголовок - позволяет задать описание к ячейке которая находится справа.

Для каждой ячейки можно задать свой цвет шрифта. Кликните правой кнопкой мыши на ячейке, выберите пункт меню **Цвет ячейки**.

![](/ut11/ut11-23.png)

## **Таблица значений для выбора в ячейке с данными**

|     |     |
| --- | --- |
| **Название** | **Описание** |
| <История продаж> | Отображение истории продаж у выбранного клиента. [Подробно](/mobilnyj-klient-na-android/dokumenty/zakaz-i-prodazha/istoriya-i-plan-prodazh) |
| <План> <Ед> | Отображает рекомендуемое количество товара к продаже. [Подробно](/integraciya-s-1s-8-ut-10/servis/nastrojka-kolonok-dokumentov) |
| <Кол-во+Бонус> <Ед> | Количество проданного в ТТ товара. Сумма содержимого полей **Кол-во** и **Бонус**. |
| <Кол-во> <Ед> | Количество проданного в ТТ товара без учета бонуса. Содержимое поля **Кол-во**. |
| <Бонус> <Ед> | Количество выписанного бонусного товара. Содержимое поля **Бонус**. |
| <Наличие> | Остаток товара по выбранному складу. |
| <НаличиеВсеСкл> | Остаток товара по **всем** складам. |
| <Склад> | Наименование выбранного склада. |
| <Наличие на основном складе> | Остаток товара на основном [складе](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/sklady). |
| <Основной склад> | Наименование основного склада. |
| <Цена> | Цена за выбранную единицу товара, без учета скидки. |
| <Цена со скидкой> | Цена за выбранную единицу с учетом скидки. |
| <Цена базовая> <Ед> | Цена за **базовую** единицу товара, без учета скидки. |
| <Сумма> | Сумма по строке (Количество\*ЦенаТовара), с учетом скидки. |
| <Остаток в точке> \| <Фэйсинг> | Мерчендайзинговая информация, содержимое полей **Остаток** и **Фейсинг** |
| <Остаток в точке> <Ед> | Мерчендайзинговая информация, содержимое поля **Остаток**. |
| <Фэйсинг> | Мерчендайзинговая информация, содержимое поля **Фейсинг**. |
| <Описание> | Описание товара. |
| <Есть фото> | Индикатор наличия выгруженных фотографий к товару. |
| **<Req\_КодРеквизита Наименование>** | Отображение значения дополнительного реквизита товара, указанные агентом. |
| <Цена закупочная> | Отображение [закупочной цены](/funkcii/raschet-nacenki). |
| <% наценки> | Отображает процент наценки или убытка. |
| <Процент скидки> | Процент скидки |
| <Сумма скидки> | Сумма скидки по строке. |
| <Сумма без скидки> | Сумма по строке (Количество\*ЦенаТовара), без учета скидки. |
| <Недостаточно меток> | Количество товара без марок. Актуально для работы с [маркированными](/funkcii/markirovannye-tovary) товарами. |

Набор значений ячейки может отличаться для разных документов.

## **Настройка Карточки товара**

Для документов **Заказ** и **Продажа** можно изменить заголовок и скрыть не используемые поля **Бонус** **Фейсинг** и **Остаток**.
