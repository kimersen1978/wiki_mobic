Какими отчетам в различных конфигурациях смотреть информацию по остатку товара, долгу клиента и список документов оснований для ПКО.

# **Конфигурация Управление торговлей 10.3 и** [**аналогичные**](https://mobi-c.ru/1cconfiguration.html)

## **Остатки и резерв товара**

**Отчеты ⇒ Запасы (склад) ⇒ Анализ доступности товаров не складах**

-   Столбец **Свободный остаток** - остаток товара на складе
-   Столбец **Зарезервировано** - резерв товара на складе

Если для товара включена детализация по характеристикам или используется настройка, Настройки агента > вкладка Номенклатура > настройка [**Детализировать на КПК по сериям**](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/nomenklatura) необходимо в отчете добавить группировку по Сериями и Характеристикам.

Если включена настройка [**Отображать остаток в разрезе организаций**](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/sklady), то **Отчеты ⇒ Запасы (склад) ⇒ Ведомость по товарам и ГТД организаций**

Столбец **Кон. Остаток** - остаток товара на складе, резервы в этом режиме не учитываются.

Поддержка остатков по складам в разрезе организаций реализована в 86 версии мобильного клиента и обработке 5.5.42.

### **Параметры влияющие на остаток на КПК**

1.Настройки агента > вкладка Номенклатура > настройка [**Учитывать резерв при расчете остатков**](/voprosy-i-otvety/ostatki-dolgi-osnovaniya)

-   Вкл = **Свободный остаток**
-   Выкл = **Свободный остаток + Зарезервировано**

2.Настройки агента **⇒** вкладка **Склады** > список доступных складов, влияет на остаток по складу **Авто**.

## **Долг клиента**

1.  **Отчеты ⇒ Продажи ⇒ Взаиморасчеты ⇒ Ведомость по взаиморасчетам с контрагентами**
2.  **Отчеты ⇒ Продажи ⇒ Взаиморасчеты ⇒ Задолженность по контрагентам**

> Внимание! В КПК долг клиента выгружается общий, без учета разделения по фирмам.

> Внимание! Долги для отчета отображаются в валюте документа. На КПК долг пересчитывается в валюту заданную в Настройки агента > вкладка Настройки > настройка [Валюта на КПК](/integraciya-s-1s-8-ut-10/agenty/nastrojki-agenta/nastrojki). Курс пересчета находится в Регистр сведений > Курсы валют.

## **Просроченный долг**

Типового отчета нет, но можно использовать Отчет по кредитной линии, неоплаченные документы и сроки оплаты

**Отчеты ⇒ Продажи ⇒ Взаиморасчеты ⇒ Отчет по кредитной линии**

## **Основания для ПКО**

Долг по документу на КПК отображается на момент выполнения полной загрузки/обновления данных. Если по документу была частичная оплата, сумма долга по документу будет с учетом частичной оплаты

-   **Отчеты > Продажи ⇒ Взаиморасчеты ⇒ Ведомость по взаиморасчетам с контрагентами**
-   **Отчеты > Продажи ⇒ Взаиморасчеты ⇒ Задолженность по контрагентам**

Выгружаются документы если в договоре стоит галка **Вести по документам расчетов с контрагентами**.

> Внимание! Долги для отчета отображаются в валюте документа. На КПК долг пересчитывается в валюту заданную в Настройки агента > вкладка Настройки > настройка [Валюта на КПК](/voprosy-i-otvety/ostatki-dolgi-osnovaniya). Курс пересчета находится в Регистр сведений > Курсы валют.

# **Конфигурация Бухгалтерия предприятия 3**

**Отчеты ⇒ Оборотно-сальдовая ведомость по счету**

Столбец **Сальдо на конец периода - Дебет** - остаток товара на складе.

### **Параметры влияющие на остаток на КПК**

1\. На КПК выгружаются остатки по следующим счетам:  
Для конфигураций БП 3 Казахстан и Таджикистан - **1310, 1320 и 1330**

Для остальных конфигураций -  **10.01, 41**

Если включена константа «ВедетсяПроизводственнаяДеятельность», то добавляется счет **43**.

2 .**Настройки агента ⇒ вкладка Склады ⇒** список доступных складов, влияет на остаток по складу **Авто**.

3\. **Настройки агента ⇒ вкладка Организации ⇒** список доступных организаций влияет на остатки по складам, детализации остатка по организациям на КПК нет.

## **Долг клиента**

**Все функции ⇒ Отчеты ⇒ Расчеты с покупателями и поставщиками**

> Внимание! В КПК долг клиента выгружается общий, без учета разделения по фирмам.

 **Все функции ⇒ Отчеты ⇒ Расчеты с покупателями и поставщиками**

> Внимание! Долги в отчете отображаются в валюте документа. На КПК долг пересчитывается в валюту заданную в Настройки агента > вкладка Настройки > настройка [Валюта на КПК.](/voprosy-i-otvety/ostatki-dolgi-osnovaniya) Курс пересчета находится в Регистр сведений > Курсы валют.

## **Конфигурация Торговля и Склад 9.2 и аналогичные**

Вы сформировали Ведомость по остаткам ТМЦ в 1С. На КПК выполнили запрос [**Полная загрузка**](/mobilnyj-klient-na-android/obmen-dannymi/polnaya-zagruzka). Сравнили остатки номенклатуры и они не совпадают. Причина в том что в настройках отчета вы указали не те параметры которые используются при выборке остатков для отправки на КПК.

Есть три параметра которые влияют на выгружаемые остатки. Все параметры задаются в настройках агента и раскиданы по разным вкладкам.

Вкладка [**Номенклатура**](/integraciya-s-1s-7-7/spravochniki/agenty/nomenklatura) параметр **Склад**.

Вкладка [**Организации**](/integraciya-s-1s-7-7/spravochniki/agenty/organizacii) список организаций

Вкладка [**Настройки**](/integraciya-s-1s-7-7/spravochniki/agenty/nastrojki) параметр **Упр. аналитика**.
